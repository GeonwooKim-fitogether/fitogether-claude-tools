# 워크플로 디자이너 (workflow-designer)

사용자가 **도메인/시스템 이름만** 줘도 (예: "PLM", "EPTS", "제조 QC"), 그 분야를 **조사해서**
비전문가도 깊게 이해할 **다층 워크플로**로 풀어주고 **자동으로 시각화**한다.
`deep-research`(조사), `superpower`(스펙 규율)와 함께 쓰면 시너지가 난다.

## 언제 쓰나
- "○○ 시스템은 어떻게 동작해?" / "○○ 워크플로 그려줘" / "○○ 만들려는데 흐름 짜줘"
- 짜고 있는 흐름에 **빠진 게 없는지** 확인하고 싶을 때, 그걸 **시각적으로** 보고 싶을 때

## 워크플로 (이 순서를 지킨다)

### 0단계: 먼저 질문한다 (추측 금지)
바로 그리지 말고, 작성 방향을 가르는 **2~4개의 핵심 질문**을 먼저 한다. `AskUserQuestion`을 쓰면 좋다.
기본 질문 세트(상황 맞춰 가감):
- **범위** — 전체 시스템인가, 특정 모듈/단계 하나인가?
- **독자 수준** — 비전문가용으로 쉽게? 실무자용으로 정밀하게?
- **목적** — 개념 이해가 목적인가, 직접 **구축**할 설계도가 목적인가?
- **기준점** — 벤치마크할 실제 제품/경쟁사/표준이 있나? (예: "Arena 같은", "FIFA EPTS 표준")
- **제약** — 규제·산업 특수성(의료기기·금융 등), 연동해야 할 시스템이 있나?

답을 받기 전엔 본 작성을 시작하지 않는다.

### 1단계: 딥리서치로 분야를 파악한다
질문 답을 바탕으로 그 도메인을 **실제 근거로** 조사한다.
- 가능하면 **`deep-research` 스킬을 사용**한다. 없으면 다중 출처 웹 검색 + 교차 확인.
- 수집할 것: **핵심 모듈/구성요소, 전문 용어(jargon), 하위 프로세스, 데이터 모델, 상태·역할,
  실제 사례, 흔한 함정/실패 지점**. 출처를 기록한다.
- 어림짐작으로 채우지 않는다 — 조사로 확인된 것만 쓴다. 모르면 "조사 필요"로 표시.

### 2단계: 구조를 계층으로 분해한다
한 장의 평면도가 아니라 **2단 구조**로 만든다.
- **Level 0 — 도메인 맵**: 모듈/구성요소가 서로 어떻게 연결되는지 (무엇이 무엇을 트리거/제약하는지)
- **Level 1 — 상세 워크플로**: 주요 모듈/시나리오마다 별도 흐름. 각 노드에
  **입력·출력·담당(액터)·상태·업무규칙**을 붙인다.

### 3단계: 완성도 점검 (누락은 반드시 표면화)
- [ ] 시작점/트리거가 명확한가
- [ ] 모든 경로가 종료 상태로 끝나는가 (dead-end 없나)
- [ ] 모든 결정 분기의 양쪽 행선지가 정의됐나
- [ ] 예외/실패 경로(오류·반려·타임아웃·재시도)가 있나
- [ ] 각 단계 담당과 단계 간 입력→출력 연결이 끊기지 않나
- [ ] 되돌림(재작업·보완) 루프, 승인/검토 지점이 빠지지 않았나
빠진 곳은 "⚠️ 누락/보완 필요: …"로 명시한다.

### 4단계: 비전문가 언어로 설명한다
- **용어집**: 등장하는 전문용어를 "쉽게 말하면 …"으로 푼다.
- 각 단계에 **"왜 이렇게 하나"**(이게 없으면 무슨 문제가 생기나)를 곁들인다.
- 시스템 빌드용이면 데이터 모델·상태·역할 표도 함께.

### 5단계: 자동으로 시각화한다 (말 안 해도)
흐름을 만들거나 고칠 때마다 **요청 없이** 시각 결과를 띄운다.
- **Artifact 도구로 렌더**해 미리보기를 바로 제공한다.
- 아티팩트 CSP가 외부 CDN을 차단하므로, HTML로 만들 땐 **외부 스크립트/폰트를 쓰지 말고**
  인라인 SVG로 직접 그리거나(권장, 확실히 렌더됨), 마크다운 아티팩트의 ```mermaid 코드블록을 쓴다.
- 도구를 못 쓰는 환경이면 repo에 `.mmd`/마크다운으로 저장하고 경로를 알려준다.

### 6단계: 반복
피드백을 받아 1~5단계를 돈다. 완성되면 `/plan/`·`docs/`에 저장할지 물어본다.

## 원칙
- **질문 먼저, 그다음 조사, 그다음 작성.** 순서를 건너뛰지 않는다.
- **근거 없는 깊이는 금물** — 조사로 뒷받침된 내용만. 모르면 표시.
- **항상 보여준다** — 흐름이 바뀌면 그림도 같이 갱신해 자동으로 띄운다.
